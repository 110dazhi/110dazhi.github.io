Git 修改最后一次提交

在实际开发中，可能会遇到以下的情景

1,版本刚一提交（commit）到仓库,突然想起漏掉两个文件还没有添加（add）。

2,版本刚一提交（commit）到仓库，突然想起版本说明写的不够全面，无法彰显本次修改的重大意义…

解决方法

执行带 –amend 选项的 commit 提交命令，Git 就会“更正”最近的一次提交。

修正完后， 按esc 然后 shift +zz 保存退出

不希望修改 就 ：q! 回车 就会保留旧的说明

如果是中文会出现乱码的话 ，就 ：q! 不保存退出 ，然后
git commit –amend -m “加上新的说明，这里可以写中文”

带m就不会进入那个让你编辑的那个界面

删除文件

使用 git checkout – README.md(文件名)

就可以恢复

删除的话 git rm yellow.jpg (文件名)

但是还会 git status 查看到这个没了

所以要用 git reset –soft HEAD~ 修改过去指针，漫天过海

git rm 文件名

  • 该命令 删除的只是工作目录和暂存区域的文件，也就是取消跟踪，在下次提交时不纳入版本管理
  • 当工作目录和暂存区域的同一个文件存在不同内容时，执行 git rm -f 文件名就可以把两个都删除。
  • 如果只删除暂存区域的文件 （保留工作目录的），那么就可以执行 git rm –cached 文件名 命令 实现目的

git rm -f 文件名 删除暂存区域和工作区域

git rm –cached test.py 只删除暂存区域

重命名文件

git mv game.py wordgame.py

  • git mv 旧文件名 新文件名
    • ren/mv 旧文件名 新文件名
    • git rm 旧文件名
    • git add 新文件名
